# Choice Properties — Replit Environment Guide

## Project Overview

**Choice Properties** is a static HTML/CSS/JavaScript rental property platform.

| Layer | Technology | Where it lives |
|---|---|---|
| Frontend | Static HTML/CSS/Vanilla JS | This repo |
| Production host | Cloudflare Pages | `choicepropertiesofficial.pages.dev` |
| Database | Supabase PostgreSQL | `tlfmwetmhthpyrytrcfo.supabase.co` |
| Auth | Supabase Auth | Same Supabase project |
| Edge Functions | Supabase Deno functions | `supabase/functions/` (deployed to Supabase) |
| Image CDN | ImageKit | `ik.imagekit.io/21rg7lvzo` |
| Email relay | Google Apps Script + Resend | External |
| Application form | External static site | `apply-choice-properties.pages.dev` |

---

## Replit's Role

Replit is used for **editing and previewing** the static site. The preview pane connects to the live Supabase backend so you can see real data while editing.

- **Local preview server:** `serve.js` — a simple Node.js HTTP server on port 5000
- **Config:** `config.js` — pre-generated from the Supabase/ImageKit values in `.replit [userenv]`
- **No local database** — all data comes from Supabase cloud
- **No migrations** — schema lives in Supabase, managed via `SETUP.sql`

### Deployment Flow

```
Edit code in Replit → Push to GitHub → Cloudflare Pages auto-builds and deploys
```

At Cloudflare build time, `generate-config.js` regenerates `config.js` from Cloudflare environment variables and injects CSP nonces, cache-bust tokens, and nav/footer components into all HTML files.

## Key Files

| File | Purpose |
|---|---|
| `serve.js` | Replit local static file server (port 5000) |
| `config.js` | Frontend config — pre-generated for Replit, regenerated by Cloudflare at deploy |
| `js/cp-api.js` | Supabase SDK client — all data access |
| `generate-config.js` | Cloudflare build-time config generator |
| `SETUP.sql` | Supabase schema reference (run on Supabase dashboard, not here) |
| `supabase/functions/` | Edge Function source (deployed to Supabase, not here) |
| `_headers` | Cloudflare Pages security headers (CSP, HSTS, etc.) |
| `_redirects` | Cloudflare Pages URL routing |

---

## What NOT to Do in Replit

- Do not run `npm install` or `npm run build` (the build script is Cloudflare-only)
- Do not replace Supabase with a local or Replit Postgres database
- Do not run `supabase` CLI commands locally
- Do not modify `config.js` by hand — edit `.replit [userenv.shared]` values instead
