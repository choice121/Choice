# Choice Properties — PERMANENT RULES FOR ALL AI TOOLS

## ⛔ STOP — READ THIS BEFORE DOING ANYTHING

This project is a **static HTML/CSS/JavaScript** site.

- **Production host:** Cloudflare Pages (deployed automatically from GitHub push)
- **Backend:** Supabase cloud — PostgreSQL, Auth, Storage, and Edge Functions
- **Replit's ONLY role:** Code editor for HTML, CSS, and JavaScript files

---

## What Replit Is Allowed To Do

- ✅ Edit `.html`, `.css`, `.js`, `.md`, `.sql`, `.ts` source files
- ✅ Read and understand the codebase
- ✅ Help write or fix frontend code

## What Replit Must NEVER Do

- ❌ Run `npm install`, `npm start`, `npm run build`, or any npm script
- ❌ Create or run `server.js`, `db/`, `migrations/`, or any Node.js server
- ❌ Install `express`, `fastify`, `pg`, `drizzle-orm`, `prisma`, or any server/DB package
- ❌ Create or connect to a local PostgreSQL, SQLite, Neon, or Replit database
- ❌ Run database migrations (`db:push`, `db:migrate`, etc.)
- ❌ Modify `supabase/` beyond editing source files
- ❌ Replace Supabase with any local or alternative backend
- ❌ "Migrate" this project to Replit hosting, Neon, or any non-Supabase backend
- ❌ Create a workflow that starts a server (there is no server to run here)
- ❌ Create `generate-config-replit.js`, `config.js` (auto-generated by Cloudflare), or similar wrappers

## Deployment Flow

```
Edit code in Replit → Push to GitHub → Cloudflare Pages auto-builds and deploys
```

Cloudflare Pages runs `npm run build`, which:
1. Validates the environment (blocks Replit/server environments)
2. Generates `config.js` from Cloudflare environment variables
3. Injects nav/footer components, CSP nonces, and cache-bust tokens into HTML

**There is no local server. There is no local database. This is intentional.**

## Key Files

| File | Purpose |
